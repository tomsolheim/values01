# Specs

This directory is the source of truth for product behavior before code is written.

Recommended workflow:

1. Write or update the feature spec in `specs/features`.
2. Add acceptance criteria in `specs/acceptance`.
3. Create failing tests that express the acceptance criteria.
4. Implement the smallest useful Laravel, Livewire, and Bootstrap code.
5. Update the spec when the intended behavior changes.

Keep specs concise, testable, and close to the language users would use.
